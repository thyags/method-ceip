# Validação de Documentação

## Objetivo

Auditar linguagem, padronização, referências, exemplos e navegabilidade dos documentos.

## Contexto

Documentação é a interface principal do framework para humanos e IAs. Inconsistência de linguagem ou navegação reduz adoção.

## Diretrizes

- Validar português do Brasil.
- Verificar estrutura mínima.
- Procurar conteúdo genérico que não ajude execução.
- Validar links internos e referências cruzadas.

## Perguntas de validação

- Todos os documentos têm título claro?
- O objetivo de cada documento explica quando ele deve ser usado?
- O contexto evita interpretação errada?
- As diretrizes são acionáveis?
- Os exemplos ajudam projetos reais?
- Os checklists são verificáveis?
- A conclusão resume uso, decisão ou próximo passo?
- A linguagem é técnica, clara e objetiva?
- Há termos inconsistentes que precisam entrar no glossário?
- `INDEX.md` cobre todos os diretórios relevantes?
- `ROADMAP.md` reflete a evolução atual?
- `README.md` dá uma rota de entrada clara?
- Há documentos muito parecidos que deveriam ser diferenciados ou consolidados?

## Exemplos

- "Melhorar qualidade" não é diretriz suficiente; deve dizer como validar.
- Um template sem exemplo de uso fica mais difícil para agentes.

## Checklist

- [ ] Estrutura documental mínima foi validada.
- [ ] Linguagem está consistente.
- [ ] Exemplos são aplicáveis.
- [ ] Links e referências foram revisados.
- [ ] Glossário foi atualizado quando necessário.

## Conclusão

Validação de documentação garante que o framework seja utilizável sem explicação verbal.
