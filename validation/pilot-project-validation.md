# Validação de Projeto Piloto

## Objetivo

Testar a CEIP em um projeto real controlado antes de adoção ampla.

## Projeto sugerido

GSA Oficina, por reunir backend, frontend, UX, banco de dados, APIs, regras de negócio, responsividade, relatórios e fluxos complexos.

## Perguntas de validação

- A IA conseguiu localizar os documentos certos?
- O Orchestrator selecionou agentes adequados?
- O Policy Engine classificou risco corretamente?
- Os brains ajudaram a decidir sem confusão?
- Quality gates bloquearam lacunas reais?
- Faltou documentação para executar a análise?
- Houve suposição indevida de regra de negócio?
- O resultado foi útil para um projeto real?

## Checklist

- [ ] Escopo do piloto foi delimitado.
- [ ] Projeto foi analisado sem alterar produção.
- [ ] Lacunas foram registradas.
- [ ] Ajustes viraram issue, ADR, RFC, pattern ou anti-pattern.
- [ ] Roadmap foi atualizado.

## Conclusão

O piloto é a prova prática da CEIP como plataforma operacional.
