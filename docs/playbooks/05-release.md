# 05 - Playbook de Release

## Objetivo

Orientar preparação, validação, comunicação e monitoramento de releases.

## Contexto

Release em software empresarial pode afetar operação, suporte, clientes, integrações e dados. O processo deve ser proporcional ao risco da mudança.

## Diretrizes

- Classificar impacto da release.
- Garantir que testes e revisões foram concluídos.
- Preparar comunicação quando houver mudança visível.
- Definir rollback ou mitigação.
- Monitorar depois da implantação.

## Fluxo

1. Consolidar escopo da release.
2. Validar checklist de QA, segurança e performance.
3. Confirmar migrações e dependências.
4. Preparar release notes.
5. Executar deploy conforme processo local.
6. Monitorar métricas, logs e feedback.
7. Registrar incidentes ou ajustes.

## Exemplos

- Release com migração de dados exige backup validado e plano de rollback.
- Release de site institucional exige checar links, SEO, performance e formulários.

## Checklist

- [ ] Escopo foi fechado.
- [ ] Testes e review foram concluídos.
- [ ] Migrações foram ensaiadas quando aplicável.
- [ ] Release notes foram preparadas.
- [ ] Rollback ou mitigação existe.
- [ ] Monitoramento pós-release foi definido.

## Conclusão

Release profissional reduz surpresa e aumenta capacidade de resposta.
