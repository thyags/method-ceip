# Exemplo Prático - Site Institucional

## Objetivo

Demonstrar uso do framework em criação ou evolução de site institucional.

## Contexto

A empresa precisa apresentar marca, serviços, cases, contato e conteúdo indexável. O site deve carregar bem, ser acessível e permitir manutenção.

## Diretrizes

- Definir público, mensagem e objetivos de conversão.
- Validar stack ou CMS existente.
- Considerar SEO, performance, acessibilidade e segurança de formulários.

## Exemplo de orquestração

1. Product Manager define objetivo do site e conversões.
2. SEO Marketing Engineer orienta estrutura de conteúdo e metadados.
3. UI Designer define linguagem visual coerente com marca.
4. Frontend UX Specialist valida navegação e responsividade.
5. Security Engineer revisa formulário de contato e dados.
6. Performance Engineer valida carregamento.
7. QA Engineer revisa links, formulários e responsividade.

## Checklist

- [ ] Público e objetivo de conversão foram definidos.
- [ ] Conteúdo é indexável.
- [ ] Formulários são seguros.
- [ ] Performance foi medida.
- [ ] Layout responde em dispositivos relevantes.

## Conclusão

Site institucional profissional combina comunicação, performance, SEO, acessibilidade e operação simples.
