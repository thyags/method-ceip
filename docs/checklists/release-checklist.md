# Checklist de Release

## Objetivo

Validar prontidão de uma release antes, durante e depois da implantação.

## Contexto

Use em releases regulares, hotfixes, deploys de site, integrações e mudanças com migração.

## Diretrizes

- Feche escopo.
- Valide risco.
- Monitore pós-deploy.

## Checklist

- [ ] Escopo da release foi confirmado.
- [ ] PRs foram revisadas e aprovadas.
- [ ] Testes necessários foram executados.
- [ ] Migrações foram revisadas e ensaiadas quando aplicável.
- [ ] Backup ou rollback foi planejado.
- [ ] Configurações e segredos foram conferidos.
- [ ] Release notes foram preparadas.
- [ ] Comunicação foi planejada.
- [ ] Monitoramento pós-deploy foi definido.
- [ ] Critério de sucesso pós-release foi registrado.

## Exemplos

Use para publicação de feature SaaS, correção crítica, atualização de site institucional ou implantação de integração.

## Conclusão

Release segura depende de preparação e acompanhamento, não apenas de deploy bem-sucedido.
