# Checklist de Frontend

## Objetivo

Validar interfaces frontend quanto a usabilidade, estados, acessibilidade, contratos e performance.

## Contexto

Use em telas administrativas, dashboards, formulários, sites institucionais, fluxos SaaS e experiências responsivas.

## Diretrizes

- Respeite design system e componentes locais.
- Cubra estados de interface.
- Não dependa do cliente para segurança crítica.

## Checklist

- [ ] Stack e padrões de componentes foram identificados.
- [ ] Jornada principal está clara.
- [ ] Estados de carregamento, vazio, erro e sucesso existem.
- [ ] Validações aparecem de forma compreensível.
- [ ] Layout é responsivo.
- [ ] Acessibilidade básica foi avaliada.
- [ ] Contrato com backend foi validado.
- [ ] Permissões sensíveis não existem apenas no frontend.
- [ ] Performance de carregamento foi considerada.
- [ ] Textos e mensagens são consistentes.

## Exemplos

Use para formulário de cadastro, painel gerencial, listagem, fluxo de checkout, landing page ou tela mobile.

## Conclusão

Frontend bom é previsível, acessível e alinhado ao comportamento real do sistema.
