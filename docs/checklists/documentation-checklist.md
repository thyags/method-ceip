# Checklist de Documentação

## Objetivo

Validar se documentos seguem o padrão CloudSix e ajudam execução, decisão ou manutenção.

## Contexto

Use em README, ADR, RFC, playbook, template, checklist, guia, documentação de API e relatório de incidente.

## Diretrizes

- Documente para o próximo responsável.
- Seja específico.
- Atualize links e índice.

## Checklist

- [ ] Documento tem título.
- [ ] Objetivo está claro.
- [ ] Contexto é suficiente.
- [ ] Diretrizes ou procedimento são acionáveis.
- [ ] Exemplos foram incluídos quando aplicável.
- [ ] Checklist é verificável.
- [ ] Conclusão resume uso ou decisão.
- [ ] Links internos estão corretos.
- [ ] Português do Brasil foi usado.
- [ ] Não há conteúdo genérico que não ajude execução.

## Exemplos

Use ao auditar este framework, ao documentar arquitetura, ao criar playbook de cliente ou ao registrar decisão.

## Conclusão

Documentação boa deve reduzir dúvida operacional e preservar contexto técnico.
