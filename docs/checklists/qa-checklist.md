# Checklist de QA

## Objetivo

Validar cobertura de testes e evidências de qualidade antes de aceite.

## Contexto

Use para features, bugs, hotfixes, releases, migrações, integrações e refatorações.

## Diretrizes

- Teste por risco.
- Cubra comportamento crítico.
- Registre lacunas.

## Checklist

- [ ] Requisitos e critérios de aceite foram revisados.
- [ ] Fluxo principal foi testado.
- [ ] Exceções e bordas foram testadas.
- [ ] Permissões foram testadas.
- [ ] Dados representativos foram usados.
- [ ] Testes automatizados foram executados quando disponíveis.
- [ ] Validação manual foi registrada quando necessária.
- [ ] Regressões relevantes foram consideradas.
- [ ] Evidências foram coletadas.
- [ ] Riscos residuais foram documentados.

## Exemplos

Use para cálculo, workflow de aprovação, checkout, login, integração, importação, relatório e alteração visual crítica.

## Conclusão

QA fecha a entrega com evidência, não com confiança informal.
