# Documentação do Framework

## Objetivo

Organizar os diretórios de documentação do CloudSix Engineering Framework e orientar quando usar cada tipo de artefato.

## Contexto

O repositório contém documentos de governança, agentes, padrões, playbooks, templates, checklists, workflows, ADRs, exemplos, arquiteturas de referência, prompts e glossário. A navegação correta evita uso incorreto de documentos.

## Diretrizes

- Use `agents` para responsabilidades de papéis.
- Use `standards` para princípios permanentes de qualidade técnica.
- Use `playbooks` para procedimentos situacionais.
- Use `templates` para criar artefatos padronizados.
- Use `checklists` para validação.
- Use `workflows` para sequência de execução.
- Use `adr` para decisões arquiteturais.
- Use `reference-architectures` para modelos conceituais.
- Use `prompts` para acionar agentes em ferramentas de IA.

## Exemplos

- Para uma feature, combine workflow, agente, template de especificação e checklist de review.
- Para um legado, combine playbook de legado, arquitetura de referência, Refactoring Specialist e QA.
