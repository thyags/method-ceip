# Rodada 2 - Documentation Engineer

## Objetivo

Padronizar linguagem, títulos, índice, referências e consistência documental.

## Contexto

Depois que a arquitetura de informação estiver validada, a documentação precisa ser uniforme para humanos e agentes de IA.

## Diretrizes

- Escrever em português do Brasil.
- Padronizar títulos e seções.
- Atualizar links internos.
- Evitar conteúdo genérico.
- Não alterar decisões técnicas sem acionar especialista.

## Perguntas de revisão

- Todos os documentos têm objetivo, contexto, diretrizes, exemplos, checklist e conclusão?
- Os títulos seguem padrão de nomenclatura?
- `INDEX.md` permite encontrar rapidamente qualquer assunto?
- `README.md` explica como começar?
- `ROADMAP.md` reflete o estado atual?
- Os documentos usam linguagem técnica, clara e objetiva?
- Há termos que precisam entrar no glossário?
- Há links internos quebrados ou ausentes?

## Saída esperada

- Lista de ajustes de padronização.
- Documentos atualizados.
- Links internos corrigidos.
- Sugestões para glossário.

## Exemplos

- Padronizar "Quality Gates", "quality gates" e "gates de qualidade" para uma forma preferencial, mantendo clareza.

## Checklist

- [ ] Linguagem foi padronizada.
- [ ] Navegação foi revisada.
- [ ] Referências internas foram atualizadas.
- [ ] Glossário foi considerado.

## Conclusão

Esta rodada torna o framework mais fácil de ler, buscar e usar por agentes.
