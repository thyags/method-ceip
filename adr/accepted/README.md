# ADRs Aceitos

## Objetivo

Armazenar decisões arquiteturais aprovadas e vigentes.

## Contexto

Uma decisão aceita orienta implementação, review, documentação e futuras decisões relacionadas.

## Diretrizes

- Manter decisão, consequências e validação visíveis.
- Atualizar links quando houver documentação complementar.
- Revisar status quando a decisão for substituída.

## Exemplos

- Decisão aceita de estratégia de tenancy.
- Decisão aceita de modelo de integração.

## Checklist

- [ ] Decisão foi aprovada.
- [ ] Consequências foram aceitas.
- [ ] Plano de validação existe.
- [ ] Dono ou área responsável foi definida.

## Conclusão

ADRs aceitos são fonte de verdade arquitetural.
