# ADRs Propostos

## Objetivo

Armazenar decisões arquiteturais ainda em avaliação.

## Contexto

Use esta pasta para decisões que precisam de review, evidência, validação ou aprovação antes de serem consideradas padrão.

## Diretrizes

- Declare claramente o problema.
- Liste alternativas reais.
- Aponte riscos e perguntas abertas.
- Relacione RFCs quando houver.

## Exemplos

- Propor nova estratégia de observabilidade.
- Propor separação de módulo crítico.

## Checklist

- [ ] Status está como proposto.
- [ ] Há perguntas abertas.
- [ ] Revisores foram indicados.
- [ ] Critério de aceite da decisão existe.

## Conclusão

ADRs propostos organizam discussão antes de comprometer a arquitetura.
