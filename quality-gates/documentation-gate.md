# Documentation Gate

## Objetivo

Validar se decisões, contratos, fluxos, operação e uso estão documentados no nível necessário.

## Quando aplicar

Aplicar em mudanças de arquitetura, API, banco, operação, processo, agente, policy, playbook, template e release.

## Critérios obrigatórios

- Documento correto foi criado ou atualizado.
- Links internos funcionam.
- Decisões relevantes têm justificativa.
- Conteúdo está em português do Brasil.

## Critérios bloqueantes

- Documento essencial ausente.
- Decisão estrutural sem ADR.
- Link interno quebrado em documento principal.
- Conteúdo contradiz a constituição.

## Evidências exigidas

Lista de documentos alterados, ADR/RFC quando aplicável, revisão de links e checklist documental.

## Agentes responsáveis

Documentation Engineer, Knowledge Curator, Chief Software Architect e agente do domínio afetado.

## Checklist

- [ ] Documento correto foi atualizado.
- [ ] Links foram verificados.
- [ ] Títulos seguem padrão.
- [ ] Exemplos existem quando úteis.
- [ ] Índice foi atualizado.

## Score mínimo

Baixo 70, médio 80, alto 85, crítico 90.

## Resultado esperado

Documentação suficiente para uso, revisão e manutenção.

## Conclusão

Documentação é parte da entrega.
