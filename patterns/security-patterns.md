# Security Patterns

## Objetivo

Agrupar padrões de segurança aplicáveis a sistemas empresariais.

## Padrões relacionados

- Menor privilégio.
- Validação de entrada.
- Segredos fora de código e documentação.
- Autorização no backend.
- Logs sem dados sensíveis.

## Quando usar

Em qualquer mudança com dados, usuários, permissões, uploads, integrações ou operações administrativas.

## Anti-patterns associados

- Segurança por obscuridade.
- Autorização apenas no frontend.
- Segredo em texto claro.

## Checklist

- [ ] Permissões foram verificadas.
- [ ] Dados sensíveis foram minimizados.
- [ ] Segredos foram protegidos.

## Conclusão

Segurança deve ser padrão de construção.
