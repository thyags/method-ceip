# Frontend Patterns

## Objetivo

Agrupar padrões de interface, estado, integração e responsividade.

## Padrões relacionados

- Componentes reutilizáveis.
- Estados explícitos de carregamento, erro, vazio e sucesso.
- Separação entre dados e apresentação.
- Acessibilidade básica.
- Layout responsivo validado.

## Quando usar

Em telas, formulários, dashboards, fluxos operacionais e componentes compartilhados.

## Anti-patterns associados

- Texto sobreposto.
- Estado de erro ausente.
- Componente visual sem comportamento definido.

## Checklist

- [ ] Estados principais foram definidos.
- [ ] Layout foi validado em mobile e desktop.
- [ ] Ações críticas têm feedback.

## Conclusão

Frontend precisa ser operável em contexto real.
