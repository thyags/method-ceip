# Testing Patterns

## Objetivo

Agrupar padrões de teste, validação e regressão.

## Padrões relacionados

- Teste de caracterização em legado.
- Teste de regressão para bugfix.
- Teste de contrato para API.
- Cenários negativos.
- Evidência manual quando automação ainda não existe.

## Quando usar

Em features, correções, refatorações, integrações, releases e incidentes.

## Anti-patterns associados

- Fluxo crítico sem teste.
- Teste que valida implementação em vez de comportamento.
- Aceite não verificável.

## Checklist

- [ ] Comportamento esperado foi descrito.
- [ ] Cenários negativos foram considerados.
- [ ] Regressão foi protegida.

## Conclusão

Teste bom protege comportamento relevante.
