# Architecture Patterns

## Objetivo

Agrupar padrões arquiteturais aplicáveis a sistemas CloudSix.

## Padrões relacionados

- Camadas bem definidas.
- Modularização por domínio.
- ADR para decisão estrutural.
- Evolução incremental.
- Integração por contratos explícitos.

## Quando usar

Quando a decisão impactar fronteiras, dados, integração, implantação ou evolução de longo prazo.

## Anti-patterns associados

- Acoplamento forte.
- Decisão não documentada.
- Reescrita ampla sem validação.

## Checklist

- [ ] O padrão resolve uma força real.
- [ ] Trade-offs foram registrados.
- [ ] A decisão tem ADR quando relevante.

## Conclusão

Padrões arquiteturais devem estabilizar evolução sem engessar tecnologia.
