# Pattern Library

## Objetivo

Catalogar padrões de design e arquitetura que podem ser úteis, com critérios de uso e riscos.

## Contexto

Padrões ajudam comunicação e solução de problemas recorrentes, mas não devem ser aplicados por moda. Cada uso deve respeitar stack, domínio e padrões locais.

## Diretrizes

- Usar padrão somente quando resolve problema real.
- Explicar quando aplicar e quando evitar.
- Adaptar à stack existente.

## Exemplos

- Adapter pode isolar fornecedor externo.
- Strategy pode representar variações explícitas de regra.

## Checklist

- [ ] Problema real foi identificado.
- [ ] Padrão reduz complexidade ou risco.
- [ ] Trade-offs foram avaliados.
- [ ] Implementação respeita padrões locais.

## Conclusão

Patterns são ferramentas, não objetivos.
