# Score System

## Objetivo

Avaliar entregas de engenharia com pontuação comparável, evidência e critérios de melhoria.

## Contexto

Checklists dizem se algo foi considerado. O Score System indica maturidade relativa da entrega e ajuda agentes a melhorar próxima iteração.

## Diretrizes

- A nota final não substitui quality gates.
- Um bloqueio crítico pode impedir aceite mesmo com pontuação alta.
- Toda nota deve ter evidência ou justificativa.
- Use a rubrica em `scoring-rubric.md`.

## Checklist

- [ ] Áreas aplicáveis foram avaliadas.
- [ ] Evidências foram registradas.
- [ ] Bloqueios foram checados.
- [ ] Nota final tem justificativa.
- [ ] Melhorias foram recomendadas.

## Conclusão

Score System transforma qualidade em feedback mensurável, sem substituir julgamento técnico.
