# Engineering Knowledge Base

## Objetivo

Servir como memória organizacional da engenharia CloudSix, registrando aprendizados, práticas recorrentes e referências internas por domínio.

## Contexto

Standards definem regras estáveis. A Knowledge Base registra aprendizados que surgem de projetos, incidentes, reviews, decisões e uso real do framework.

## Diretrizes

- Registrar conhecimento aplicável, não opinião solta.
- Conectar aprendizados a ADRs, RFCs, incidents, reviews ou examples quando possível.
- Evitar duplicação com standards; se virar regra permanente, promover para `docs/standards`.
- Manter conteúdo em português do Brasil.

## Áreas

- `architecture`
- `backend`
- `frontend`
- `security`
- `testing`
- `performance`
- `database`
- `ai`
- `business`

## Exemplos

- Aprendizado de incidente de integração vai para `knowledge/backend` ou `knowledge/security`.
- Padrão recorrente de UX em ERP vai para `knowledge/frontend`.

## Checklist

- [ ] Aprendizado tem origem ou contexto.
- [ ] Conteúdo é reutilizável.
- [ ] Links internos foram adicionados.
- [ ] Não há duplicação com standard existente.

## Conclusão

Knowledge Base transforma experiência em vantagem operacional reutilizável.
