# Knowledge DevOps

## Objetivo

Reunir conhecimento sobre deploy, ambientes, observabilidade, rollback, incidentes e operação.

## Conteúdos esperados

- Estratégias de deploy.
- Planos de rollback.
- Observabilidade.
- Gestão de ambientes.
- Incidentes e postmortems.

## Checklist

- [ ] Conhecimento tem evidência.
- [ ] Não expõe segredos.
- [ ] Relaciona-se com playbooks e gates.
- [ ] Pode ser aplicado de forma agnóstica.

## Conclusão

DevOps na CEIP é disciplina operacional, não ferramenta específica.
