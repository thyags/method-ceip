# Knowledge DevOps

## Objetivo

Reunir conhecimento sobre deploy, ambientes, observabilidade, rollback, incidentes e operação.

## Conteúdos esperados

- Estratégias de deploy.
- Planos de rollback.
- Observabilidade.
- Gestão de ambientes.
- Incidentes e postmortems.

## Relação com outros módulos

- `quality-gates/release-gate.md` define critérios de liberação.
- `docs/playbooks/05-release.md` orienta preparação de release.
- `docs/playbooks/06-incidente-producao.md` orienta resposta a incidente.
- `memory/lesson-learned-memory.md` registra aprendizado recorrente.

## Exemplos de conhecimento útil

- Checklist de rollback que funcionou em release crítica.
- Padrão de monitoramento para fluxo operacional.
- Lição aprendida sobre janela de deploy.
- Sinal de alerta que antecipou incidente.

## Checklist

- [ ] Conhecimento tem evidência.
- [ ] Não expõe segredos.
- [ ] Relaciona-se com playbooks e gates.
- [ ] Pode ser aplicado de forma agnóstica.
- [ ] Pode orientar uma decisão operacional futura.

## Conclusão

DevOps na CEIP é disciplina operacional, não ferramenta específica.
