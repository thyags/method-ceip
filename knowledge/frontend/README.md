# Knowledge Base - Frontend

## Objetivo

Registrar aprendizados sobre UX, UI, estados, acessibilidade e performance frontend.

## Contexto

Interfaces empresariais precisam ser eficientes, claras e tolerantes a erro. Esta base reúne padrões observados no uso real.

## Diretrizes

- Registrar contexto de usuário e tarefa.
- Relacionar design system ou componentes quando aplicável.
- Promover padrões recorrentes para standards se estabilizarem.

## Conhecimento inicial

- Estado vazio deve orientar próxima ação.
- Erro recuperável deve preservar dados digitados.
- Tabelas operacionais precisam de filtros, paginação ou limite quando houver alto volume.

## Exemplos

- Em CRM, listas devem favorecer comparação e ação rápida.
- Em oficina, telas mobile devem priorizar campos essenciais e alvos de toque.

## Checklist

- [ ] Jornada do usuário foi considerada.
- [ ] Estados foram descritos.
- [ ] Acessibilidade foi avaliada.

## Conclusão

Conhecimento frontend deve melhorar produtividade e reduzir erro do usuário.
