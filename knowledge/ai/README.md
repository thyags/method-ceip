# Knowledge Base - IA

## Objetivo

Registrar aprendizados sobre uso de IA, prompts, avaliação, segurança, automação e agentes.

## Contexto

IA pode ampliar produtividade, mas exige governança. Esta base registra padrões e limitações observadas.

## Diretrizes

- Registrar caso de uso, dados, avaliação e limites.
- Não incluir dados sensíveis em exemplos.
- Diferenciar automação determinística de saída probabilística.

## Conhecimento inicial

- Prompt sem avaliação não é solução confiável.
- IA deve declarar incerteza em tarefas de decisão.
- Ações sensíveis geradas por IA precisam de revisão humana ou controle equivalente.

## Exemplos

- Classificador de tickets deve ser avaliado com amostras reais anonimizadas.
- Assistente de documentação deve seguir templates do framework.

## Checklist

- [ ] Caso de uso justifica IA.
- [ ] Dados foram minimizados.
- [ ] Avaliação foi definida.
- [ ] Fallback existe.

## Conclusão

Conhecimento de IA deve aumentar utilidade sem reduzir governança.
