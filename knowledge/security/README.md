# Knowledge Base - Segurança

## Objetivo

Registrar aprendizados de segurança, privacidade, autorização, integrações e IA.

## Contexto

Segurança evolui com incidentes, reviews e novos tipos de integração. Esta base mantém aprendizados práticos.

## Diretrizes

- Não registrar segredos ou detalhes sensíveis.
- Relacionar risco, mitigação e evidência.
- Promover regra recorrente para constitution ou standards quando necessário.

## Conhecimento inicial

- Exportações devem ser tratadas como ações sensíveis.
- Webhooks precisam de validação, idempotência e rastreabilidade.
- Prompts de IA devem minimizar dados e registrar limitações.

## Exemplos

- Um token encontrado em log deve gerar incidente e revisão de logging.
- Permissão ausente em endpoint deve acionar teste de autorização.

## Checklist

- [ ] Dados sensíveis não foram expostos.
- [ ] Mitigação foi descrita.
- [ ] Risco residual foi registrado.

## Conclusão

Conhecimento de segurança deve reduzir reincidência de falhas.
