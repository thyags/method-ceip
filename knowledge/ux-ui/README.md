# Knowledge UX/UI

## Objetivo

Reunir conhecimento sobre experiência, interface, acessibilidade, responsividade e consistência visual.

## Conteúdos esperados

- Fluxos de usuário.
- Padrões de componentes.
- Estados de interface.
- Acessibilidade.
- Responsividade.

## Relação com outros módulos

- `quality-gates/frontend-gate.md` valida experiência e estados.
- `policy-engine/UX_UI_POLICIES.md` define políticas de mudança visual.
- `patterns/frontend-patterns.md` reúne padrões de interface.
- `memory/lesson-learned-memory.md` registra aprendizados de uso.

## Exemplos de conhecimento útil

- Estado vazio que reduziu erro operacional.
- Fluxo de formulário que diminuiu retrabalho.
- Padrão de feedback para ação destrutiva.
- Critério de responsividade usado em dashboard crítico.

## Checklist

- [ ] O aprendizado melhora uso real.
- [ ] Há exemplo visual ou comportamento esperado quando útil.
- [ ] Estados de erro, vazio e carregamento foram considerados.
- [ ] O conteúdo respeita padrões locais de cada projeto.
- [ ] O aprendizado pode ser aplicado sem impor stack.

## Conclusão

UX/UI na CEIP deve ajudar operação real e reduzir fricção.
