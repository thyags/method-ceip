# Knowledge Base - Testes

## Objetivo

Registrar aprendizados sobre validação, regressão, dados de teste e automação.

## Contexto

Testes eficientes seguem risco e comportamento. Esta base registra estratégias úteis por tipo de entrega.

## Diretrizes

- Conectar teste a requisito ou risco.
- Registrar dados necessários para reproduzir cenários.
- Distinguir validação manual de automação.

## Conhecimento inicial

- Legado sem cobertura exige testes de caracterização antes de refatoração.
- Bug recorrente deve gerar teste de regressão.
- Migração de dados precisa de validação antes/depois.

## Exemplos

- Cálculo financeiro deve usar casos históricos.
- Integração deve ter cenários de timeout, retry e duplicidade.

## Checklist

- [ ] Cenários críticos foram identificados.
- [ ] Dados são representativos.
- [ ] Evidência foi registrada.

## Conclusão

Conhecimento de testes aumenta confiança sem depender apenas de cobertura percentual.
