# Recipe - Criar API

## Objetivo

Orientar criação de API ou endpoint com contrato, segurança, testes e observabilidade.

## Contexto

Use para APIs internas, públicas, integrações, webhooks e automações.

## Diretrizes

- Identificar estilo e padrões existentes.
- Documentar contrato.
- Validar autorização e erros.
- Planejar testes de contrato.

## Passos

1. Confirmar requisito e consumidores.
2. Identificar stack, roteamento e padrões de API.
3. Definir contrato de entrada, saída e erro.
4. Avaliar autenticação, autorização e rate limits quando aplicável.
5. Implementar conforme padrões locais.
6. Criar testes de contrato e cenários de erro.
7. Registrar documentação e observabilidade.

## Agentes

Business Analyst, Backend Engineer, API Integration Engineer, Security Engineer, QA Engineer, Documentation Engineer.

## Exemplos

- Endpoint de consulta de pedidos.
- Webhook de atualização de pagamento.

## Checklist

- [ ] Contrato foi documentado.
- [ ] Autorização foi validada.
- [ ] Erros são previsíveis.
- [ ] Testes cobrem contrato.
- [ ] Logs permitem diagnóstico.

## Conclusão

API pronta é contrato confiável, não apenas rota funcionando.
