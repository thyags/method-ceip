# Recipe - Criar Relatório

## Objetivo

Orientar criação de relatório confiável, performático e rastreável.

## Contexto

Relatórios podem influenciar decisões financeiras, operacionais e comerciais. Dados e filtros precisam ser definidos com precisão.

## Diretrizes

- Confirmar pergunta de negócio.
- Definir fonte de dados e filtros.
- Validar performance e consistência.
- Documentar limitações.

## Passos

1. Confirmar objetivo e usuário do relatório.
2. Definir métricas, filtros, período e agrupamentos.
3. Mapear fontes de dados e regras de cálculo.
4. Avaliar volume e performance.
5. Criar visualização ou exportação.
6. Testar com dados representativos.
7. Documentar fórmula, filtros e limitações.

## Agentes

Business Analyst, Database Architect, Backend Engineer, Frontend UX Specialist, Performance Engineer, QA Engineer.

## Exemplos

- Relatório financeiro mensal.
- Relatório de estoque mínimo.

## Checklist

- [ ] Métricas foram definidas.
- [ ] Fonte de dados foi validada.
- [ ] Performance foi considerada.
- [ ] Dados de teste são representativos.
- [ ] Limitações foram documentadas.

## Conclusão

Relatório confiável precisa de semântica de dados tão clara quanto a apresentação.
