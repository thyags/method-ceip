# Recipe - Criar Dashboard

## Objetivo

Orientar criação de dashboard operacional, gerencial ou analítico.

## Contexto

Dashboards devem apoiar decisão, acompanhamento e ação. Eles podem exigir agregação, filtros, permissões e performance.

## Diretrizes

- Confirmar métricas com negócio.
- Definir fonte de dados.
- Evitar métricas decorativas sem ação.
- Validar performance e UX.

## Passos

1. Confirmar público, decisão suportada e métricas.
2. Mapear fonte de dados e atualização.
3. Definir filtros, granularidade e permissões.
4. Desenhar UX com estados e responsividade.
5. Avaliar performance de consultas e payload.
6. Testar dados, bordas e permissões.
7. Documentar métricas e limitações.

## Agentes

Business Analyst, Product Manager, Database Architect, Frontend UX Specialist, UI Designer, Performance Engineer, QA Engineer.

## Exemplos

- Dashboard de vendas SaaS.
- Painel operacional de oficina.

## Checklist

- [ ] Métricas têm definição.
- [ ] Fonte de dados foi validada.
- [ ] Permissões foram avaliadas.
- [ ] Performance foi medida quando necessário.
- [ ] Estados de interface foram definidos.

## Conclusão

Dashboard deve responder perguntas reais com dados confiáveis.
