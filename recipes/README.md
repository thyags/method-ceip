# Engineering Recipes

## Objetivo

Fornecer receitas práticas para tarefas recorrentes de engenharia, orientando agentes e pessoas com passos, validações e gates.

## Contexto

Receitas ajudam IA a executar tarefas comuns sem esquecer governança, contexto e qualidade. Elas não substituem análise da stack e do domínio.

## Diretrizes

- Toda receita começa por contexto e stack.
- Não inventar requisitos.
- Usar agentes e gates adequados.
- Registrar ADR/RFC quando a receita gerar decisão estrutural.

## Exemplos

- Para criar API, use API Integration Engineer, Backend Engineer, Security Engineer e QA Engineer.
- Para criar relatório, use Business Analyst, Database Architect, Performance Engineer e QA Engineer.

## Checklist

- [ ] Receita aplicável foi escolhida.

## Conclusão

Recipes transformam tarefas comuns em execução guiada e verificável.
