# Decision Framework

## Objetivo

Organizar decisões técnicas e operacionais por contexto, alternativas, trade-offs, risco e evidência.

## Contexto

O documento raiz `DECISION_FRAMEWORK.md` define o método. Este diretório pode receber árvores, modelos e exemplos adicionais.

## Processo

1. Definir problema.
2. Levantar restrições.
3. Listar alternativas.
4. Comparar trade-offs.
5. Classificar risco.
6. Decidir com justificativa.
7. Registrar ADR, RFC ou policy quando aplicável.

## Checklist

- [ ] Problema foi descrito.
- [ ] Alternativas foram comparadas.
- [ ] Trade-offs foram registrados.
- [ ] Risco foi classificado.
- [ ] Registro formal foi criado quando necessário.

## Conclusão

Decisão boa deixa rastros suficientes para revisão futura.
