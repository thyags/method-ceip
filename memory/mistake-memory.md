# Mistake Memory

## Objetivo

Registrar erros recorrentes para evitar repetição.

## Conteúdo permitido

- Erro observado.
- Sinal de detecção.
- Impacto.
- Correção recomendada.
- Anti-pattern relacionado.

## Limites

- Não expor pessoas.
- Não registrar dado sensível.
- Não transformar culpa em processo.

## Checklist

- [ ] O erro tem evidência.
- [ ] O impacto foi descrito.
- [ ] A correção é acionável.
- [ ] Anti-pattern foi atualizado quando necessário.

## Conclusão

Memória de erro existe para melhorar sistema, não apontar culpados.
