# Architecture Memory

## Objetivo

Registrar decisões arquiteturais recorrentes, restrições e aprendizados estruturais.

## Conteúdo permitido

- Contexto de decisão.
- Trade-offs observados.
- Restrições arquiteturais.
- Relação com ADRs.

## Limites

- Decisão relevante deve estar em ADR, não apenas neste arquivo.
- Não registrar detalhes internos sensíveis quando resumo basta.

## Checklist

- [ ] O aprendizado aponta ADR quando necessário.
- [ ] Alternativas foram registradas.
- [ ] Risco residual foi descrito.
- [ ] O registro é agnóstico de tecnologia quando possível.

## Conclusão

Memória arquitetural reduz repetição de discussões já resolvidas.
