# Pattern Memory

## Objetivo

Registrar padrões que funcionaram bem em projetos e os contextos em que se aplicam.

## Conteúdo permitido

- Nome do pattern.
- Contexto de sucesso.
- Benefícios observados.
- Limites de uso.
- Link para `patterns/`.

## Limites

- Não promover padrão sem evidência.
- Não apresentar solução como universal.

## Checklist

- [ ] Contexto de aplicação está claro.
- [ ] Benefícios foram observados.
- [ ] Limites foram registrados.
- [ ] Pattern correspondente foi atualizado.

## Conclusão

Pattern bom inclui quando usar e quando evitar.
