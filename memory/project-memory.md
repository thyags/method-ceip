# Project Memory

## Objetivo

Registrar contexto reutilizável de projetos sem vazar informações sensíveis.

## Conteúdo permitido

- Tipo de projeto.
- Arquitetura geral.
- Restrições relevantes.
- Riscos recorrentes.
- Decisões documentadas.

## Limites

- Não registrar credenciais, dados pessoais, contratos, URLs internas sensíveis ou detalhes comerciais sem necessidade.
- Usar identificadores genéricos quando possível.

## Checklist

- [ ] O contexto ajuda futuras análises.
- [ ] Informações sensíveis foram removidas.
- [ ] ADRs e RFCs foram referenciados.
- [ ] Riscos foram atualizados.

## Conclusão

Memória de projeto deve acelerar entendimento sem comprometer confidencialidade.
