# CEIP Profiles

## Objetivo

Definir perfis de adoção que ajustam profundidade de governança, quantidade de gates e artefatos exigidos.

## Perfis Iniciais

| Perfil | Uso |
| --- | --- |
| Startup | Baixa cerimônia, foco em velocidade e aprendizado |
| Enterprise | Governança completa, rastreabilidade e aprovação |
| Legacy | Modernização incremental, caracterização e risco |
| Open Source | Transparência, contribuição e documentação pública |
| Solo Developer | Simplicidade e baixo overhead |
| Agency | Reuso entre clientes e separação forte de contexto |
| Consultoria | Diagnóstico, auditoria e handoff |
| SaaS | Produto contínuo, métricas, UX e release recorrente |
| ERP | Processos, permissões, dados e operação crítica |

## Team Profiles

- Equipe pequena: menos documentos obrigatórios, gates proporcionais ao risco.
- Empresa média: governança padrão, Product Intelligence e Runtime obrigatórios.
- Enterprise: validações completas, aprovação formal e trilha de auditoria.

## Maturity Levels

| Nível | Nome | Descrição |
| --- | --- | --- |
| 1 | Prompt | Uso pontual de instruções |
| 2 | Method | Método documentado |
| 3 | Framework | Módulos integrados |
| 4 | Engineering Platform | Runtime, Workspace, Installer e governança |
| 5 | Engineering OS | Evolução, marketplace, profiles e automação contínua |

## Conclusão

Profiles reduzem carga cognitiva e adaptam a CEIP ao tamanho, risco e maturidade de cada organização.
