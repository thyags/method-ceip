# Recipe: Gerar PRD

## Quando usar

Quando já existem discovery, contexto de negócio e requisitos iniciais.

## Passos

1. Validar se Discovery Brief existe.
2. Validar Requirements Map.
3. Preencher `prd-template.md`.
4. Marcar lacunas em vez de inventar respostas.
5. Submeter ao PRD Engine.
6. Acionar Policy Engine para obrigatoriedade de RFC, ADR ou aprovação.

## Saída

- PRD completo ou PRD com lacunas explícitas.

## Gate

- PRD sem escopo, fora de escopo, critérios de aceite ou MVP não pode seguir para arquitetura.
