# Recipe: Criar Backlog

## Quando usar

Quando PRD, MVP e features já existem.

## Passos

1. Listar epics.
2. Quebrar features em stories.
3. Anexar critérios de aceite.
4. Ordenar por valor, risco e dependência.
5. Separar MVP, V1 e futuro.
6. Encaminhar para Product Planning Engine.

## Saída

- Backlog inicial.
- Dependências.
- Handoff para Architecture e Orchestrator.

## Gate

- Story sem feature e critério de aceite deve ser bloqueada.
