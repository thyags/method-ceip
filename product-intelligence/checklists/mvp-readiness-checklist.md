# MVP Readiness Checklist

## Validação

- [ ] Objetivo central do MVP está claro.
- [ ] Usuários atendidos foram definidos.
- [ ] Funcionalidades essenciais foram separadas de futuras.
- [ ] Fora de escopo foi declarado.
- [ ] Métrica de sucesso existe.
- [ ] Riscos e premissas foram registrados.
- [ ] Condição para avançar para V1 foi definida.

## Bloqueios

- [ ] MVP não depende de feature futura para entregar valor.
- [ ] MVP não exige arquitetura incompatível com evolução.
- [ ] MVP não omite regra crítica de negócio.
