# Documentation Rules

## Objetivo

Definir quando documentação deve ser criada ou atualizada.

## Contexto

CEIP depende de memória persistente.

## Regras

- Mudança de arquitetura exige ADR.
- Mudança de escopo relevante exige RFC.
- Mudança de contrato exige documentação de API ou spec.
- Mudança operacional exige playbook ou release notes.
- Novo módulo CEIP exige atualização de INDEX.
- Nova regra repetitiva exige policy.

## Exemplos

- Criar `brains/` exige atualizar INDEX, ROADMAP e validation.

## Checklist

- [ ] Documento afetado foi identificado.
- [ ] Atualização foi feita.
- [ ] Índice foi revisado.

## Conclusão

Documentação é parte da entrega.
