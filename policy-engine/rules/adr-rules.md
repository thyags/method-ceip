# ADR Rules

## Objetivo

Definir quando ADR é obrigatório.

## Contexto

ADR registra decisão arquitetural relevante.

## Regras

- Mudança de arquitetura exige ADR.
- Mudança de fonte de verdade exige ADR.
- Nova dependência estrutural pode exigir ADR.
- Mudança difícil de reverter exige ADR.
- Decisão que afeta múltiplos módulos exige ADR.

## Exemplos

- Adotar CEIP como escopo estratégico gerou ADR aceito.

## Checklist

- [ ] Decisão é arquitetural?
- [ ] Impacto é amplo?
- [ ] Reversibilidade foi avaliada?
- [ ] ADR foi criado ou justificativa foi registrada?

## Conclusão

ADR torna arquitetura explicável.
