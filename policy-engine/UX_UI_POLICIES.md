# UX/UI Policies

## Objetivo

Governar mudanças visuais, usabilidade, acessibilidade e experiência de usuário em alinhamento com o Product Experience System.

## Contexto

Mudanças visuais podem afetar operação, erro humano, responsividade, acessibilidade e percepção de qualidade do produto.

## Diretrizes

- Alteração visual com impacto funcional aciona UX/UI e QA.
- Interface relevante aciona Product Experience System antes de UX/UI/Frontend.
- Estados de interface devem ser considerados.
- Acessibilidade básica é obrigatória.
- Interface operacional não deve priorizar decoração sobre clareza.
- Visual Quality Score é obrigatório quando a interface for relevante.

## Políticas

- Toda tela nova exige estados de loading, vazio, erro e sucesso quando aplicável.
- Formulário deve preservar dados em erro recuperável.
- Ação destrutiva exige comunicação proporcional ao risco.
- Dashboard, tabela, formulário e página de detalhe relevantes exigem Product Experience Gate.
- Benchmarks não podem ser usados como cópia visual.

## Exemplos

- Dashboard administrativo deve favorecer leitura, filtros e ação rápida.
- Tabela operacional deve favorecer comparação, status acessível, ações contextuais e densidade controlada.

## Checklist

- [ ] Jornada foi descrita.
- [ ] PXS foi aplicado quando obrigatório.
- [ ] Estados foram definidos.
- [ ] Responsividade foi avaliada.
- [ ] Acessibilidade foi considerada.
- [ ] Visual Quality Score foi calculado quando obrigatório.

## Conclusão

UX/UI Policies protegem experiência operacional, prevenção de erro e padrão visual premium.
