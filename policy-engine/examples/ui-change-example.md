# Exemplo - Alteração Visual

## Objetivo

Mostrar como classificar mudanças visuais e UX/UI.

## Contexto

Mudanças visuais podem ser baixo risco quando cosméticas ou médio/alto quando alteram fluxo.

## Diretrizes

- Diferenciar cosmético de funcional.
- Acionar Product Experience System quando a interface for relevante.
- Acionar UX/UI quando houver mudança de jornada.
- Validar acessibilidade e responsividade.
- Calcular Visual Quality Score quando houver impacto de produto.

## Aplicação

- Tipo: alteração visual.
- Risco: baixo se cosmético, médio se altera interação.
- Agentes obrigatórios: Product Experience System quando relevante, Frontend UX Specialist, UI Designer, QA quando houver fluxo.
- Documentos: experience brief, critérios visuais, checklist frontend e Visual Quality Score quando aplicável.
- Gates: Product Experience, Frontend, QA, Documentation quando aplicável.

## Exemplos

Trocar texto de botão é baixo risco. Mudar fluxo de aprovação visual é médio/alto. Criar dashboard operacional exige PXS e Visual Quality Score.

## Checklist

- [ ] Impacto funcional foi avaliado.
- [ ] PXS foi aplicado quando a interface era relevante.
- [ ] Estados de interface foram revisados.
- [ ] Responsividade foi testada.
- [ ] Acessibilidade foi considerada.
- [ ] Visual Quality Score foi registrado quando aplicável.

## Conclusão

UI não é sempre baixo risco; depende do impacto no fluxo.
