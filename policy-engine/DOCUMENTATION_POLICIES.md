# Documentation Policies

## Objetivo

Governar atualização documental da CEIP e de projetos que a adotam.

## Contexto

Mudanças sem documentação tornam o sistema dependente da memória de pessoas ou modelos.

## Diretrizes

- Mudança relevante deve atualizar documentação.
- ADR/RFC deve ser criado quando aplicável.
- Índice deve ser atualizado para novos módulos.
- Documentação deve ser acionável.

## Políticas

- Novo módulo da CEIP exige entrada no INDEX.
- Nova policy exige relação com Policy Engine.
- Nova recipe exige agentes, gates e validações.
- Novo aprendizado recorrente deve ir para knowledge ou memory.

## Exemplos

- Criar um engine sem atualizar layers e index é falha documental.

## Checklist

- [ ] Documentos afetados foram atualizados.
- [ ] Índice foi atualizado.
- [ ] ADR/RFC foi avaliado.
- [ ] Conteúdo é acionável.

## Conclusão

Documentation Policies preservam navegabilidade e memória.
