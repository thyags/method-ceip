# Benchmark Library

## Objetivo

Documentar princípios de experiência observáveis em produtos digitais de referência, sem copiar identidade, layout, marca ou componentes proprietários.

## Regra

Benchmarks são fonte de análise, não de reprodução. O PXS deve extrair padrões de clareza, densidade, hierarquia, estados, fluxo e produtividade, adaptando-os ao domínio e ao design system local.

## Produtos analisados

- `linear.md`
- `notion.md`
- `figma.md`
- `github.md`
- `vercel.md`
- `stripe.md`
- `atlassian.md`
- `hubspot.md`
- `clickup.md`
- `airtable.md`

## Método de análise

Para cada benchmark, observar:

- uso de espaço;
- organização de ações;
- hierarquia;
- estados;
- tabelas e listas;
- formulários;
- navegação;
- feedback;
- densidade;
- sensação de produto premium.

## Checklist

- [ ] O benchmark foi usado como princípio, não como cópia.
- [ ] O domínio do projeto foi considerado.
- [ ] A decisão foi adaptada à stack e ao design system local.
- [ ] A justificativa foi registrada.

## Conclusão

Benchmark bom inspira critérios. Benchmark mal usado cria cópia sem identidade.
