# Benchmark: GitHub

## Objetivo da análise

Extrair princípios de navegação por repositório, rastreabilidade, discussão e revisão.

## Princípios úteis

- Histórico e autoria são visíveis.
- Estados de revisão são claros.
- Comentários aparecem próximos do contexto.
- Listas densas continuam escaneáveis.
- Ações críticas exigem intenção explícita.

## Aplicação na CEIP

Use esses princípios em auditorias, reviews, logs, ADRs, RFCs, tarefas técnicas e fluxos com rastreabilidade.

## Cuidados

- Não copiar layout de repositório quando o domínio for diferente.
- Não expor complexidade técnica para usuário não técnico.
- Preservar clareza para perfis operacionais.

## Checklist

- [ ] Histórico e decisão são rastreáveis.
- [ ] Discussões ficam perto do contexto.
- [ ] Estados são claros.
- [ ] Ações críticas têm proteção.
