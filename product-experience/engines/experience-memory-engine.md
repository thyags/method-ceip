# Experience Memory Engine

## Objetivo

Registrar, classificar e reutilizar padrões de experiência aprovados sem misturar dados de clientes no CEIP Core.

## Entradas

- Tela ou fluxo aprovado.
- Visual Quality Score.
- Decisão de layout.
- Evidências permitidas.
- Limitações de reutilização.

## Processamento

1. Verificar se não há dados sensíveis.
2. Classificar o padrão: dashboard, tabela, formulário, sidebar, detalhe, fluxo ou navegação.
3. Registrar problema resolvido e contexto.
4. Registrar por que funcionou.
5. Registrar condições em que não deve ser reutilizado.
6. Atualizar Workspace local.

## Saídas

- Registro de experiência.
- Referência local para projetos futuros.
- Aprendizado candidato para Knowledge Layer quando for genérico.

## Checklist

- [ ] Dados sensíveis foram removidos.
- [ ] O padrão tem justificativa.
- [ ] O score ou evidência sustenta o registro.
- [ ] Limites de reutilização foram descritos.
- [ ] O Core não recebeu contexto privado do cliente.

## Conclusão

Experience Memory Engine permite aprendizado visual sem comprometer privacidade, marca ou contexto específico de cliente.
