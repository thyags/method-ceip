# Design DNA

## Objetivo

Definir os atributos de percepção que devem orientar a experiência de produtos CloudSix.

## Atributos

| Atributo | Como aparece na interface |
| --- | --- |
| Confiança | Estados claros, dados legíveis, ações previsíveis e feedback objetivo |
| Velocidade | Fluxos curtos, navegação direta, busca eficiente e ações contextuais |
| Organização | Grids, agrupamentos coerentes, filtros bem posicionados e hierarquia estável |
| Tecnologia | Precisão visual, consistência, automação útil e linguagem objetiva |
| Inteligência | Antecipação de necessidades, prevenção de erro e priorização de informação |
| Simplicidade | Menos ruído visual, menos repetição e menor carga cognitiva |
| Maturidade | Decisões visuais sóbrias, acessíveis e adequadas ao domínio empresarial |

## Regras

- Cada tela deve reforçar pelo menos três atributos do Design DNA.
- Telas críticas devem priorizar confiança, organização e prevenção de erro.
- Telas de produtividade devem priorizar velocidade, densidade e ação contextual.
- Telas analíticas devem priorizar comparação, legibilidade e evidência.
- Telas institucionais podem ser mais expressivas, mas não devem parecer genéricas.

## Perguntas de revisão

- O produto parece confiável para uma operação real?
- A tela ajuda o usuário a agir mais rápido?
- A composição organiza a informação ou apenas a exibe?
- A experiência comunica inteligência sem exagero visual?
- A simplicidade preserva as informações necessárias?

## Checklist

- [ ] A tela reforça o DNA esperado.
- [ ] O domínio do produto foi considerado.
- [ ] A experiência não parece CRUD genérico.
- [ ] A interface transmite maturidade operacional.

## Conclusão

O Design DNA mantém consistência de percepção entre produtos, mesmo quando stacks, marcas e domínios variam.
