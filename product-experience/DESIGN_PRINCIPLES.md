# Design Principles

## Objetivo

Definir princípios de design para produtos empresariais premium, operacionais e agnósticos de tecnologia.

## Princípios

### Clareza antes de estilo

Uma tela bonita que exige interpretação excessiva falhou. Clareza de objetivo, estrutura e ação vem antes de preferências estéticas.

### Hierarquia antes de densidade

Interfaces empresariais podem conter muitos dados, mas a leitura precisa ter ordem. Densidade sem hierarquia vira ruído.

### Produtividade antes de decoração

Elementos visuais devem reduzir tempo, erro ou esforço. Decoração sem função deve ser removida.

### Consistência antes de novidade

Uma solução visual nova só deve ser criada quando padrões existentes não resolvem o problema.

### Contexto antes de componente

Não escolha componente antes de entender tarefa, frequência de uso, risco, volume de dados e dispositivo.

### Evidência antes de gosto

Decisões visuais devem ser justificadas por tarefa, domínio, acessibilidade, benchmark, teste, padrão local ou score.

### Simplicidade com capacidade

Simplificar não significa esconder complexidade necessária. Significa organizar complexidade para que ela seja operável.

## Anti-patterns

- Criar card para tudo.
- Usar botões grandes para ações secundárias.
- Resolver hierarquia apenas com cor.
- Usar dashboard decorativo sem decisão acionável.
- Criar formulário longo sem agrupamento e progressão.
- Esconder estado de erro para manter a tela limpa.
- Usar padrão de landing page em sistema operacional.

## Checklist

- [ ] A decisão melhora clareza ou produtividade.
- [ ] O layout respeita a frequência de uso.
- [ ] A hierarquia está evidente sem depender de explicação.
- [ ] A consistência local foi priorizada.
- [ ] Há justificativa para componentes novos.

## Conclusão

Design na CEIP é uma disciplina operacional: reduzir ambiguidade, aumentar produtividade e proteger a percepção de qualidade do produto.
