# Visual Quality Gate Checklist

## Objetivo

Validar critérios obrigatórios antes de aprovar interface relevante.

## Critérios obrigatórios

- [ ] Product Experience System foi aplicado.
- [ ] CloudSix Design Language foi considerada.
- [ ] Layout foi justificado por tarefa e contexto.
- [ ] Componentes seguem padrões locais ou exceção registrada.
- [ ] Estados principais foram especificados.
- [ ] Acessibilidade básica foi avaliada.
- [ ] Responsividade foi verificada.
- [ ] Visual Quality Score foi calculado.
- [ ] Bloqueios não compensatórios foram avaliados.
- [ ] Evidências foram registradas.

## Bloqueios

- [ ] Texto ilegível ou sobreposto.
- [ ] Ação principal ausente em fluxo operacional.
- [ ] Estado de erro ausente em operação crítica.
- [ ] Layout quebrado em viewport obrigatório.
- [ ] Interface sem relação com requisito.
- [ ] Uso de benchmark como cópia visual.

## Conclusão

O gate visual protege o padrão premium da CloudSix antes que a interface avance.
