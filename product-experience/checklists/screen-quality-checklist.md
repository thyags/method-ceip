# Screen Quality Checklist

## Objetivo

Validar qualidade mínima de uma tela, componente composto ou fluxo visual.

## Checklist

- [ ] Objetivo da tela é evidente.
- [ ] Ação principal é clara.
- [ ] Hierarquia visual tem no máximo três níveis dominantes por região.
- [ ] Layout não depende de cards desnecessários.
- [ ] Informações relacionadas estão agrupadas.
- [ ] Botões secundários não competem com ação principal.
- [ ] Estados de loading, vazio, erro, sucesso e disabled existem quando aplicáveis.
- [ ] Textos não sobrepõem nem quebram de forma incoerente.
- [ ] Interface funciona em viewports obrigatórios.
- [ ] Informação crítica não depende apenas de cor.
- [ ] A tela parece produto comercial, não template genérico.

## Conclusão

Uma tela pronta deve ser legível, acionável, responsiva, consistente e adequada ao domínio.
