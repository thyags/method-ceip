# Rodada 05 - Code Review

## Objetivo

Avaliar qualidade técnica, coesão, manutenibilidade, clareza e riscos de implementação ou documentação técnica.

## Entradas

Diffs, padrões locais, checklists, arquitetura e evidências de teste.

## Critérios

- Simplicidade.
- Coesão.
- Baixo acoplamento.
- Testabilidade.
- Compatibilidade com padrões existentes.

## Checklist

- [ ] A mudança é compreensível.
- [ ] O escopo é controlado.
- [ ] Não há duplicação indevida.
- [ ] Testes protegem comportamento.
- [ ] Dívidas foram registradas.

## Saída esperada

Achados por severidade, com recomendação objetiva e bloqueios quando necessário.

## Conclusão

Code review é controle de qualidade e manutenção futura.
